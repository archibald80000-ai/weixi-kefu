# weixi-kefu

`weixi-kefu` 是一个面向微信公众号客服系统的规划与架构设计仓库。

它不是一个已经完成的运行时代码仓库，而是一套从场景约束、技术选型、MVP 范围、系统架构到开发计划的完整前期设计资料，适合做项目立项、方案评审和实现前的结构化准备。

## 仓库定位

- 聚焦微信公众号客服场景
- 以规划文档、架构草案和实施路径为核心
- 适合前期设计和团队对齐，不是可直接启动的生产系统

## 仓库包含什么

- 对现有闲鱼客服项目的拆解参考
- 微信公众号客服约束与场景分析
- 技术选型与路线对比
- MVP 范围定义
- 系统架构草案
- 模块拆分与多 Agent 协作建议
- 风险清单与开发阶段计划

## 适合谁

- 想做公众号客服系统，但还处在架构设计阶段的人
- 需要先做方案评审、范围界定和技术对齐的团队
- 想把已有客服 Agent 思路迁移到微信生态的人
